# Data-Driven Deity Structure Spawning

This document describes the reworked, configurable deity structure-spawning system: how it
works, how to configure it, and its current limitations.

## What changed

The mod previously spawned each deity only at its own dedicated `god_temple_<deity>` worldgen
structure (one structure instance per deity, tracked in `GodWorldState`). That legacy system is
**unchanged and still works**.

Layered on top is a new, additive **tag/tier spawning system**: deities can now appear at
**existing vanilla or modded structures** based on tiered categories, domain tags, deity weights,
cooldowns, per-structure spawn budgets, progression gates, and data-driven placement/alignment.

### Why server-side detection (no spawn-list event)

Forge 1.20.1 exposes **no structure spawn-list hook**. The old `StructureSpawnListGatherEvent`
(1.16-era) was removed once structure mob spawns moved to datapack `spawn_overrides`, and
`spawn_overrides` cannot express "roll a weighted, gated, budgeted deity encounter."

Instead, `StructureSpawnManager` polls online players on a throttled server tick
(`detection_interval_ticks`, default every 5 s) and asks
`ServerLevel.structureManager().getAllStructuresAt(pos)` / `getStructureAt(pos, structure)`
which structure (and bounding box) the player currently stands in. This avoids `ChunkLoadEvent`
spam and only does cheap chunk-local lookups. All state is persisted in world `SavedData`.

## The decision stack

For each structure a player is standing in, the manager evaluates a deterministic weighted
condition stack (logged in full when `debug_logging = true`):

```
candidate deities  = deities whose whitelist/domain-tags/any-structure match this structure
selected deity     = weighted pick over candidates (deity weights)
tier               = per-structure override  ->  datapack tier tag  ->  deity default_tier
final chance       = base_chance(tier)
                       x structure tier modifier (reserved, 1.0)
                       x biome/dimension modifier
                       x deity weight modifier (clamped)
                       x world difficulty scalar
                     forced to 0 if progression gates fail
spawn if           = deterministicRoll(structureKey, gameDay) < final chance
                     AND budget available AND not on cooldown / exhausted
```

The roll is seeded per `(structure instance, in-game day)` so it is reproducible and debuggable
and cannot be spammed by re-entering the structure on the same day.

## Configuration

### Per-deity (datapack JSON — `data/<ns>/.../gods/<deity>.json`, `structure` block)

ForgeConfigSpec cannot express arbitrary per-deity arrays of arbitrary structure ids, so
per-deity structure config lives in the deity JSON (datapack/modpack overridable), where deity
data already lives:

```jsonc
"structure": {
  // ---- legacy dedicated-temple fields (unchanged) ----
  "template_id": "spells_n_gods:god_temple_deus",
  "biome_tag": "spells_n_gods:has_structure/god_temple",
  "search_radius": 10000,
  "min_dist_from_spawn": 1000,
  "size": 16,

  // ---- new data-driven tag/tier spawning ----
  "default_tier": "rare",                 // tier for tag-matched structures
  "disable_default": false,               // true = only the explicit whitelist counts
  "domain_tags": [                        // opt into structures by tag
    "#spells_n_gods:wisdom_structures"
  ],
  "whitelist": [                          // explicit structures + selection weights
    { "id": "minecraft:stronghold", "weight": 25 },
    { "id": "minecraft:village_plains", "weight": 10 },
    { "id": "somemod:library_ruin", "weight": 40 }
  ],
  "placement": "nearest_safe_floor",      // see placement strategies below
  "default_alignment": "neutral",         // see alignments below
  "alignment_overrides": {                // per-tag / per-structure behaviour variance
    "#spells_n_gods:death_structures": "hostile",
    "#spells_n_gods:wisdom_structures": "neutral"
  }
}
```

A `whitelist` entry always beats a tag-default weight. `disable_default: true` makes a deity
spawn **only** at its explicit whitelist (ignoring domain tags and `allow_any_structure`).

### Global / tiers (TOML — `config/spells_n_gods/structure_spawns.toml`)

Generated by Forge on first run. Example:

```toml
[general]
    enabled = true
    allow_any_structure = false           # permit deities at ANY detected structure
    debug_logging = false                  # log a full decision breakdown
    detection_interval_ticks = 100
    default_placement = "nearest_safe_floor"
    first_discovery_announcements = true
    difficulty_scalar_peaceful = 0.0
    difficulty_scalar_easy = 0.75
    difficulty_scalar_normal = 1.0
    difficulty_scalar_hard = 1.25
    structure_tier_overrides = [           # beat a deity's default tier
        "minecraft:ancient_city=legendary",
        "minecraft:stronghold=rare",
        "minecraft:village_plains=common"
    ]
    dimension_modifiers = ["minecraft:the_nether=1.2"]

[tier_common]
    base_chance = 0.03
    cooldown_days = 20
    max_active_deities = 1
    requires_dragon_killed = false
    required_advancement = ""
    required_dimension = ""
    min_favor = 0

# ... tier_uncommon, tier_rare ...

[tier_legendary]
    base_chance = 0.20
    cooldown_days = -1                     # -1 = permanent exhaustion (one deity ever)
    max_active_deities = 1
    requires_dragon_killed = true
```

### Domain & tier tags (datapack — `data/spells_n_gods/tags/worldgen/structure/`)

Built-in domain tags (modded structures opt in by adding themselves to these tags, no code
required):

`#spells_n_gods:ocean_structures`, `forge_structures`, `death_structures`,
`wisdom_structures`, `war_structures`, `nature_structures`, `underworld_structures`.

Optional tier-hint tags let a datapack categorise structures without touching config:
`#spells_n_gods:tier/{common,uncommon,rare,legendary}`.

> Note: this mod's namespace is `spells_n_gods` (underscore), so tags are
> `#spells_n_gods:...` rather than the generic `#spellsngods:...` used in design notes.

## Placement strategies

`center_of_structure`, `nearest_safe_floor`, `nearest_air_above_floor`,
`random_valid_position`, `shrine_anchor` (uses a `MonumentBlock`/`RuinedIdolBlock` as anchor if
present, else falls back). Every resolved position is verified safe: solid floor, breathable
non-fluid air column tall enough for the deity, never inside blocks, lava, or the void.

## Alignments (behaviour variance)

`hostile`, `neutral`, `passive`, `challenge`, `dormant`, `summon_only`. Combat-active alignments
(`hostile`/`neutral`/`challenge`) arrive with full boss AI; the others arrive inert until
provoked. Alignment can vary per structure/tag via `alignment_overrides` (e.g. a war deity is
`neutral` at war camps but `hostile` at a rival's libraries).

## Progression gates

Supported on any tier: `requires_dragon_killed` (vanilla `minecraft:end/kill_dragon`
advancement), `required_advancement` (any advancement id), `required_dimension` (player's
current dimension), and `min_favor` (favor with the candidate deity). Unknown/invalid gate
values **fail safe** — they block the spawn and log a warning rather than crashing.

## Persistence & safety

`StructureEncounterState` (world `SavedData`) stores one compact record per structure instance,
keyed by `dimension @ structureId @ originChunk`, tracking cooldown end, total spawns, and the
set of active deity UUIDs (the spawn budget). Records are pruned automatically once their
cooldown elapses and no deities remain active; permanently-exhausted records are retained.
Budget slots are released on the `LivingDeathEvent` of a tag-spawned deity. All logic is
server-side and multiplayer-safe (single-threaded tick, deterministic per-day rolls).

## Debug commands

- `/spells_n_gods spawning here` — structures at your position and which domain tags they match.
- `/spells_n_gods spawning encounters` — list persisted encounter records.
- `/spells_n_gods spawning prune` — force-prune expired records.

Set `debug_logging = true` to log the full decision breakdown for every evaluation.

## Tests

Pure decision logic (tiers, deity profiles, weighted selection, chance stack, cooldown/budget,
config validation) lives in `com.otectus.spells_n_gods.spawning.logic` with **no Minecraft
dependencies** and is unit-tested in `src/test` (`SpawnLogicTest`), covering: tier defaults,
override/whitelist precedence, domain-tag resolution, weighted selection, cooldowns, permanent
exhaustion, spawn budgets, invalid-config warnings, gate-blocked spawns, and state
reconstruction. Run with `./gradlew test`.

## Limitations / future work

- **Advancement/journal**: the mod has no journal/codex system; first-discovery currently sends
  a chat message + sound. A custom advancement criterion trigger is a clean future extension.
- **Alignment** maps onto the existing boss AI as combat-active vs inert; finer faction/targeting
  behaviour is left as an extension point.
- `required_dimension` checks the player's **current** dimension (no historical "has visited"
  tracking).
- Budget slots are released on death; a deity that despawns without dying keeps its slot until
  its cooldown elapses (then the record is prunable).
